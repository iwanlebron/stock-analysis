部署到 Supabase 需要将项目迁移到 **Supabase Edge Functions**。
由于 Supabase Edge Functions 主要支持 Deno (TypeScript/JavaScript) 运行环境，我们需要将现有的 Go 后端逻辑重写为 TypeScript。

这是一个非常好的架构升级，可以让项目完全 Serverless 化，免去维护服务器的成本。

### 迁移计划 (Go -> Supabase Edge Functions)

1.  **初始化 Supabase 环境**：
    *   创建 `supabase/` 目录结构。
    *   配置 `supabase/config.toml`。

2.  **创建 Edge Function (`fear-greed`)**：
    *   创建 `supabase/functions/fear-greed/index.ts` 作为入口。
    *   该函数将同时处理 API 请求和 HTML 页面返回 (Server-Side Rendering)。

3.  **移植核心逻辑 (Go -> TypeScript)**：
    *   **数据获取 (`yahoo.ts`)**：将 `internal/data/yahoo.go` 中的 Yahoo API 请求逻辑移植到 TS。
    *   **指标计算 (`indicators.ts`)**：将 `internal/calc/indicators.go` 和 `engine.go` 中的数学公式（RSI, MACD, 滚动百分位等）移植到 TS。

4.  **前端集成**：
    *   将 `index.html` 的内容内嵌到 Edge Function 中，作为根路径 (`/`) 的响应。
    *   `/fear-greed` 路径则作为 API 接口响应 JSON 数据。

5.  **本地验证**：
    *   使用 Deno 本地运行该 Function 进行测试，确保计算结果与 Go 版本一致。
